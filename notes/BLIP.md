BLIP 实现视觉问答，本质上把它建模成了一个**多模态理解**任务，而非常规的分类任务，流程分两步。

### 第一步：视觉编码

首先，BLIP 的视觉编码器（ViT）将输入图像转换成一系列视觉特征序列。这个序列会作为后面文本解码器的“上下文”。

### 第二步：核心——多模态混合解码器

这是实现 VQA 的关键。BLIP 的文本解码器非常灵活，叫 **多模态混合编码器-解码器 (MED)**。它通过自注意力掩码来控制工作模式，对于 VQA 使用的是**解码器模式**。

解码器的输入是一个完整的句子序列，具体构造如下：

**[图像特征序列] [编码器输出的问题特征] [解码器生成的答案序列]**

解码过程很直观：

-   解码器先利用图像特征和问题编码之间的**交叉注意力**，把图文信息深度融合，理解问题在问什么。
-   接着，它通过**因果自注意力掩码**，从左到右逐词生成答案，生成当前词时只能看到之前的内容。

举个例子，输入图是一张“猫在桌子上”的图片，问题是“What is on the table?”，解码器的输入构造就是：`[图像特征] [What, is, on, the, table, ?] [cat]`，然后模型会接着预测下一个词，直到生成结束符。

### 关键对比：Captioning vs VQA

这种架构的灵活性在于：

-   **生成描述时**：输入是 `[图像特征] [文本解码器开始符]`，之后纯靠因果自注意力生成描述。
-   **解答问题时**：输入变为 `[图像特征] [问题] [答案]`，利用因果自注意力生成答案。图像和问题部分会做双向注意力以充分理解，而答案部分则用单向掩码生成。

因此，BLIP 没有为 VQA 单独设计一个分类头，而是利用了同一个强大的解码器，在生成任务中学到的能力，直接生成自由形式的答案，这就是它实现 VQA 的方式。

# BLIP 的编码器与解码器数量

## 简短回答

BLIP 共有 **2 组参数**，但承担 **3 种功能角色**：

| 组件 | 数量 | 说明 |
|------|------|------|
| 图像编码器（ViT） | **1 个** | 独立编码图像 |
| 多模态文本编码器 | **1 个**（共享权重） | 用于 ITC、ITM |
| 多模态文本解码器 | **1 个**（共享权重） | 用于 LM（生成） |

> 关键点：文本编码器和文本解码器**共享同一套 Transformer 权重**，只是注意力掩码不同。

---

## 详细架构

```
┌────────────────────────────────────────────────────────┐
│                    BLIP 整体架构                         │
├────────────────────────────────────────────────────────┤
│                                                        │
│  ① 图像编码器（ViT）── 1个                              │
│     图像 → [Patch Embed → Transformer × N层] → 图像特征  │
│                                                        │
│  ② MED（Multimodal Mixture of Encoder-Decoder）── 1套权重 │
│     同一组 Transformer 层，通过切换注意力掩码实现三种模式：  │
│                                                        │
│     模式A：单模态文本编码器（ITC用）                      │
│     模式B：图文融合编码器（ITM用）                        │
│     模式C：图文融合解码器（LM用）                         │
│                                                        │
└────────────────────────────────────────────────────────┘
```

---

## MED 的三种工作模式

### 模式 A：单模态文本编码器（用于 ITC）

```
文本 → [双向 Self-Attention] → 文本向量
         （无 Cross-Attention，不看图像）
```
- 目的：得到纯文本表示，与图像向量做对比学习

### 模式 B：图文融合编码器（用于 ITM）

```
文本 → [双向 Self-Attention] → [Cross-Attention ← 图像特征] → 融合表示
```
- 目的：判断图文是否匹配（二分类）

### 模式 C：图文融合解码器（用于 LM）

```
文本 → [因果 Self-Attention（单向）] → [Cross-Attention ← 图像特征] → 预测下一个词
```
- 目的：根据图像生成文本描述（自回归）

---

## 为什么能共享权重？

三种模式的**网络结构完全相同**（Self-Attn + Cross-Attn + FFN），区别仅在于：

| | 模式 A（ITC） | 模式 B（ITM） | 模式 C（LM） |
|---|---|---|---|
| Self-Attention 掩码 | 双向（全可见） | 双向（全可见） | **因果（只看左边）** |
| Cross-Attention | ❌ 关闭 | ✅ 开启 | ✅ 开启 |
| 输出用途 | 对比向量 | 匹配/不匹配 | 生成下一个 token |

> 仅靠**切换注意力掩码**和**是否启用 Cross-Attention**，同一套参数就能在编码器和解码器之间切换。

---

## 总结图示

```
BLIP = 1 个图像编码器 + 1 个"变形"Transformer（MED）

                         ┌─ 模式A：文本编码器（ITC）
                         │
图像 → [ViT 编码器] ──→──┼─ 模式B：图文编码器（ITM）   ← 共享同一套权重
                         │
                         └─ 模式C：图文解码器（LM）
```

**所以：1 个图像编码器 + 1 个文本编码器 + 1 个文本解码器 = 功能上 3 个，参数上只有 2 套。**
