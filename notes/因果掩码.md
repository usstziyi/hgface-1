# 训练时因果注意力如何并行？——逐步推演

## 核心困惑

> "因果注意力不是只能看左边吗？那后面的 token 不是要等前面的算完？怎么能并行？"

**关键洞察：训练时所有 token 已经存在（teacher forcing），不需要"生成"。因果掩码只是限制"谁能看谁"，而不是限制"谁先算谁后算"。**

---

## 用一个具体例子推演

假设答案部分有 3 个 token：**A B C**（比如 "三 个 人"）

---

### 第一步：所有 token 同时输入

```
输入（同时存在，不需要等待）：

  位置1: A    位置2: B    位置3: C
   ↓           ↓           ↓
  [0.2]      [0.5]      [0.8]     ← 各自的 embedding（同时准备好）
  [0.1]      [0.3]      [0.6]
  [0.4]      [0.7]      [0.9]
```

> 训练时 A、B、C 都是**已知的**，直接查表得到 embedding，无需任何等待。

---

### 第二步：并行计算 Q、K、V

对**所有位置同时**做线性变换（一次矩阵乘法）：

$$Q = X \cdot W_Q, \quad K = X \cdot W_K, \quad V = X \cdot W_V$$

```
X = [A; B; C]  (3×d 矩阵)    ← 三个 token 一起

Q = X · W_Q = [q_A; q_B; q_C]   ← 一次矩阵乘法，同时得到3个query
K = X · W_K = [k_A; k_B; k_C]   ← 一次矩阵乘法，同时得到3个key
V = X · W_V = [v_A; v_B; v_C]   ← 一次矩阵乘法，同时得到3个value
```

> ✅ 完全并行，没有任何先后依赖。

---

### 第三步：计算注意力分数矩阵（一次矩阵乘法）

$$S = Q \cdot K^T$$

```
        k_A    k_B    k_C
q_A  [  2.1    1.3    0.7  ]   ← A 对 A,B,C 的原始分数
q_B  [  1.8    2.5    1.1  ]   ← B 对 A,B,C 的原始分数
q_C  [  0.9    1.6    3.0  ]   ← C 对 A,B,C 的原始分数
```

> ✅ 一次 $3 \times d$ 乘 $d \times 3$ 的矩阵乘法，**所有分数同时算出**。

---

### 第四步：施加因果掩码（关键！）

```
因果掩码 M：
     A    B    C
A  [ 0    -∞   -∞ ]   ← A 只能看 A
B  [ 0     0   -∞ ]   ← B 能看 A, B
C  [ 0     0    0 ]   ← C 能看 A, B, C

施加后：S_masked = S + M

        k_A    k_B    k_C
q_A  [  2.1   -∞     -∞  ]   ← B、C 的分数被设为 -∞
q_B  [  1.8    2.5   -∞  ]   ← C 的分数被设为 -∞
q_C  [  0.9    1.6    3.0 ]   ← 全部保留
```

> ⚡ **这一步就是"因果"的全部含义！** 不是"等前面算完"，而是**把不该看的分数设为 $-\infty$**。

---

### 第五步：Softmax（逐行独立，并行）

$$\alpha_{ij} = \text{softmax}(S_{\text{masked}})$$

```
        k_A    k_B    k_C
q_A  [  1.0    0      0   ]   ← A 只关注自己（-∞ → e^(-∞) = 0）
q_B  [  0.33   0.67   0   ]   ← B 关注 A 和 B
q_C  [  0.15   0.30   0.55]   ← C 关注 A、B、C
```

> ✅ 三行 softmax **互相独立**，同时计算。

---

### 第六步：加权求和得到输出（一次矩阵乘法）

$$O = \alpha \cdot V$$

```
output_A = 1.0·v_A + 0·v_B + 0·v_C       = v_A
output_B = 0.33·v_A + 0.67·v_B + 0·v_C   = 混合(A,B)
output_C = 0.15·v_A + 0.30·v_B + 0.55·v_C = 混合(A,B,C)
```

> ✅ 一次 $3 \times 3$ 乘 $3 \times d$ 的矩阵乘法，**三个输出同时得到**。

---

## 完整流程一览

```
输入 A, B, C（同时存在）
       ↓
  ┌────────────────────────────────────────────┐
  │  Q = X·Wq    （一次矩阵乘法，并行）          │
  │  K = X·Wk    （一次矩阵乘法，并行）          │
  │  V = X·Wv    （一次矩阵乘法，并行）          │
  │                                            │
  │  S = Q·K^T   （一次矩阵乘法，并行）          │
  │  S = S + M   （加掩码，逐元素，并行）         │
  │  α = softmax(S)（逐行独立，并行）            │
  │  O = α·V     （一次矩阵乘法，并行）          │
  └────────────────────────────────────────────┘
       ↓
输出 output_A, output_B, output_C（同时得到）
```

**整个过程中没有任何"等待"，全部是矩阵运算，GPU 一次算完。**

---

## 为什么不需要串行？关键对比

### 推理时（必须串行）：

```
要生成第3个token C，但C是什么？→ 不知道！
必须先算出 A → 再算出 B → 才能算出 C
因为 C 的输入 embedding 取决于 B 的生成结果
```

### 训练时（可以并行）：

```
C 是什么？→ 已知！就是训练数据里的 "人"！
A、B、C 的 embedding 同时就有了
C 不需要"等"B 被生成出来，因为 B 已经在那了

C 只需要在注意力计算时"不看未来"→ 用掩码遮住就行
```

---

## 一个直觉类比

| | 推理（串行） | 训练（并行） |
|---|---|---|
| 类比 | **考试写作文**：必须从第一个字开始写，写完一个字才知道下一个写什么 | **老师批改作文**：作文已经写好了，老师一眼看完全文，只是规定"评第2个字时假装没看到第3个字" |
| 关键区别 | 内容未知，必须逐步生成 | 内容已知，只需限制"视线方向" |

---

## 那多层 Transformer 呢？

```
第1层：A, B, C 并行计算 → 得到 A', B', C'
       ↓（层间有依赖，必须等第1层算完）
第2层：A', B', C' 并行计算 → 得到 A'', B'', C''
       ↓
第3层：...
```

- **层内**：所有位置并行（掩码限制视线）
- **层间**：必须逐层（第2层输入依赖第1层输出）

> 但这对 BERT（双向）和 GPT（因果）都一样！层间串行是所有 Transformer 的共同特征，不是因果注意力特有的。

---

## 一句话总结

> **训练时因果注意力的并行原理：所有 token 已经存在（teacher forcing），Q/K/V 同时算出，注意力分数一次矩阵乘法得到，因果性仅通过在分数矩阵上"加 $-\infty$ 掩码"实现——遮住未来，而非等待过去。整个过程全是矩阵运算，GPU 一把算完，无需逐 token 推进。**
